<span>12 Aug, 2019</span></ul></div></div><div class=text><div class=imgWrap><a href="https://practice.geeksforgeeks.org/courses?ref=gfg"><img src=https://media.geeksforgeeks.org/wp-content/cdn-uploads/20210701154555/GOL_sale-_article-min.png alt=Geek-O-Lympics class="no-zoom-in-cursor eventBannerCSS"></a></div><p>The main issue with <strong>D</strong>istance Vector <strong>R</strong>outing (DVR) protocols is Routing Loops, since <a href=https://www.geeksforgeeks.org/dynamic-programming-set-23-bellman-ford-algorithm/>Bellman-Ford Algorithm</a> cannot prevent loops. This routing loop in DVR network causes Count to Infinity Problem. Routing loops usually occur when any interface goes down or two-routers send updates at the same time.<p><strong>Counting to infinity problem:</strong><br><img src=https://media.geeksforgeeks.org/wp-content/uploads/counting-to-infinity.jpg alt width=415 height=121 class="aligncenter size-full wp-image-245647"><p>So in this example, the Bellman-Ford algorithm will converge for each router, they will have entries for each other. B will know that it can get to C at a cost of 1, and A will know that it can get to C via B at a cost of 2.<p><img src=https://media.geeksforgeeks.org/wp-content/uploads/counting-to-infinity2.jpg alt width=434 height=136 class="aligncenter size-full wp-image-245650"><br>If the link between B and C is disconnected, then B will know that it can no longer get to C via that link and will remove it from it&#8217;s table. Before it can send any updates it&#8217;s possible that it will receive an update from A which will be advertising that it can get to C at a cost of 2. B can get to A at a cost of 1, so it will update a route to C via A at a cost of 3. A will then receive updates from B later and update its cost to 4. They will then go on feeding each other bad information toward infinity which is called as <strong>Count to Infinity problem</strong>.<h3>Solution for Count to Infinity problem:-</h3><p><strong>Route Poisoning:</strong><br>When a route fails, distance vector protocols spread the <em>bad news</em> about a route failure by poisoning the route. Route poisoning refers to the practice of advertising a route, but with a special metric value called Infinity. Routers consider routes advertised with an infinite metric to have failed. Each distance vector routing protocol uses the concept of an actual metric value that represents infinity. RIP defines infinity as 16. The main disadvantage of poison reverse is that it can significantly increase the size of routing announcements in certain fairly common network topologies.<br><img src=https://media.geeksforgeeks.org/wp-content/uploads/route-poison.jpg alt width=532 height=156 class="aligncenter size-full wp-image-245657"></p><br><div id=AP_G4GR_5></div><div id=d6f45c53-ff16-4cc5-8b91-eb6ee7037116 class=_ap_apex_ad><script>var adpushup = window.adpushup  = window.adpushup || {que:[]};adpushup.que.push(function() {adpushup.triggerAd("d6f45c53-ff16-4cc5-8b91-eb6ee7037116");});</script></div><br><p><strong>Split horizon:</strong><br>If the link between B and C goes down, and B had received a route from A , B could end up using that route via A. A would send the packet right back to B, creating a loop. But according to Split horizon Rule, Node A does not advertise its route for C (namely A to B to C) back to B. On the surface, this seems redundant since B will never route via node A because the route costs more than the direct route from B to C.<br>Consider the following network topology showing Split horizon-<br><img src=https://media.geeksforgeeks.org/wp-content/uploads/route-poison2.jpg alt width=536 height=128 class="aligncenter size-full wp-image-245666"><p></br><ul><li>In addition to these, we can also use split horizon with route poisoning where above both technique will be used combinely to achieve efficiency and less increase the size of routing announcements.<li>Split horizon with Poison reverse technique is used by Routing Information Protocol (RIP) to reduce routing loops. Additionally, <strong>Holddown timers</strong> can be used to avoid the formation of loops. Holddown timer immediately starts when the router is informed that attached link is down. Till this time, router ignores all updates of down route unless it receives an update from the router of that downed link. During the timer, If the down link is reachable again, routing table can be updated.</ul><p></br><p><strong>References:</strong><br><a href=https://en.wikipedia.org/wiki/Distance-vector_routing_protocol#Count-to-infinity_problem target=_blank>https://en.wikipedia.org/wiki/Distance-vector_routing_protocol#Count-to-infinity_problem</a><br><a href=https://en.wikipedia.org/wiki/Route_poisoning target=_blank>https://en.wikipedia.org/wiki/Route_poisoning</a><br><a href=https://en.wikipedia.org/wiki/Split_horizon_route_advertisement>https://en.wikipedia.org/wiki/Split_horizon_route_advertisement</a><p>This article is contributed by <strong>Akash Sharan</strong>.
